1Deploy to Vercel (10 minutes)

What you need:

  • Vercel account (free - sign up at vercel.com)
  • Command line access

Commands to run:

# Install Vercel CLI
npm i -g vercel

# Login to Vercel (opens browser window)
vercel login

# Deploy to production
vercel --prod
What happens: Vercel will build your app and give you a URL like gigmate-xxx.vercel.app. Write this URL down!

2Add Password Protection (3 minutes)

In Vercel Dashboard:

  1. Click on your project name
  2. Go to Settings
  3. Click Deployment Protection in the left sidebar
  4. Enable "Password Protection"
  5. Set a password (example: GigMate2025Beta!)
  6. Click Save
‚ö†Ô∏è Important: Save this password! You'll send it to beta testers.

3Configure Environment Variables (5 minutes)

In Vercel Dashboard:

  1. Go to Settings ‚Üí Environment Variables
  2. Add these variables (get values from your Supabase dashboard):
Variable Name Where to Get Value
VITE_SUPABASE_URL Supabase Dashboard ‚Üí Settings ‚Üí API ‚Üí Project URL
VITE_SUPABASE_ANON_KEY Supabase Dashboard ‚Üí Settings ‚Üí API ‚Üí anon/public key
VITE_STRIPE_PUBLISHABLE_KEY Stripe Dashboard ‚Üí Developers ‚Üí API keys ‚Üí Publishable key (use test key)
VITE_GOOGLE_MAPS_API_KEY Optional - Leave blank for now
After adding variables: Go to Deployments tab and click "Redeploy" on the latest deployment.
